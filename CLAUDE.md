# LaTeX 项目题目提取与结构初始化工作流

## 1. 任务概述
本项目的工作流旨在自动化构建 LaTeX 目录结构。系统需要在指定的基础目录下新建一个项目文件夹，并从多个指定的源目录中提取特定格式的内容（仅限题目部分）和图片资源，重命名并汇总至这个新建的项目文件夹中，同时按照要求控制排版密度与垂直间距。

## 2. 核心参数
在执行任务时，必须先向用户确认以下四个参数：
- **`BASE_DIR` (基础目录)**：用户指定的父目录路径。
- **`PROJECT_NAME` (项目名称)**：将在 `BASE_DIR` 下创建的新文件夹的名称。
- **`SOURCE_DIRS` (源目录列表)**：包含所有原始 `.md` 文件、`.tex` 文件及对应 `figures/` 文件夹的路径数组或列表。
- **`PROBLEMS_PER_PAGE` (每页题数)**：指定生成的 LaTeX 章节文件中，每页排版的题目数量（用于自动插入分页符）。

## 3. 执行步骤 (Workflow)

### 步骤一：初始化目标项目结构
1. 访问并进入指定的 `BASE_DIR`。
2. 在该目录下创建一个名为 `PROJECT_NAME` 的新文件夹，并将此文件夹设为当前工作根目录（Workspace）。
3. 在工作根目录内，创建基础文件夹结构：
   - 创建 `chapters/` 目录。
   - 创建 `figures/` 目录。
4. 在工作根目录内创建并初始化主文件 `main.tex`。

### 步骤二：批量读取与内容提取
1. 遍历 `SOURCE_DIRS` 列表中的每一个目录。
2. 在当前遍历的目录中，扫描 Markdown 文件 (`.md`) 以及 LaTeX 文件夹中的 `.tex` 文件。
3. **精准提取**：解析这些文本文件，仅提取其中的“题目”部分（如带编号的问题描述），忽略非题目类的正文内容。
4. 记录每个提取出的题目最初来源于哪个源目录，以备后续抓取关联图片。

### 步骤三：合并与生成章节文件及分页控制
1. 针对步骤二中提取出的所有题目，按照其所属的“章节名称”进行分类合并。
2. 在工作根目录的 `chapters/` 目录下，以该章节名称创建对应的 `.tex` 文件（例如：`chapter_name.tex`）。
3. 将提取出的题目内容按照规范的 LaTeX 格式写入到对应的章节 `.tex` 文件中。**在每写入一道题目后，必须紧接着自动插入 `\vfill` 命令，以确保题目在页面中垂直均匀分布。**
4. **排版与分页控制**：在写入题目内容时，系统需进行计数。每当连续写入的题目数量达到 `PROBLEMS_PER_PAGE` 的设定值时，必须在当前的 `\vfill` 之后自动插入 LaTeX 分页命令（例如 `\newpage` 或 `\clearpage`），然后再继续写入下一题。

### 步骤四：图片跨目录处理与迁移（防冲突机制）
1. 根据提取出来的题目内容，回溯定位到其原本所属源目录下的 `figures/` 文件夹，寻找依赖图片。
2. **重命名与防覆盖规则**：将所需的图片名称，严格按照其最终在目标文件中对应的**题目序号**进行全局重命名（例如：无论原图叫什么，只要对应 1.1 题，就重命名为 `fig_1_1.png` 或类似规范）。
3. 将重命名后的图片文件迁移到工作根目录的 `figures/` 文件夹中。
4. 确保新生成的 `.tex` 章节文件中的图片引用路径（如 `\includegraphics{figures/fig_...}`）与重命名后的图片名称完全一致。

### 步骤五：推送到 GitHub 远程仓库
1. 在所有做题本项目生成完成后，将整个 `BASE_DIR` 目录推送到远程 Git 仓库。
2. 远程仓库地址为 `git@github.com:wjkuacs/Problem-Solving-Book.git`。
3. 使用 `git add -A` 暂存所有新增和修改的文件。
4. 使用 `git commit` 提交更改，提交信息应包含新增的做题本
